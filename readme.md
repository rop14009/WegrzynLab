    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-eukaryote-non-model-transcriptome-annotation-pipeline-entap" class="anchor" href="#eukaryote-non-model-transcriptome-annotation-pipeline-entap" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Eukaryote Non-Model Transcriptome Annotation Pipeline (enTAP)</h1>

<h2><a id="user-content-introduction" class="anchor" href="#introduction" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Introduction</h2>

<p>The software tool, enTAP, is a set of Python scripts designed to improve the accuracy and speed of functional annotations for de novo transcriptome (gene space) assemblies of non-model Eukaryotic species.</p>

<p>The scripts in this package enable the following:</p>

<ul>
<li>Integrates BLAST-style search results from up to three unique databases (and selects the most optimal annotation)</li>
<li>Identifies informative and uninformative annotations consistently</li>
<li>Identifies contaminants through optional filters for fungal, bacterial, and insect annotations.</li>
<li>Generates BLAST2GO compatible XML files to facilitate Gene Ontology term assignments</li>
<li>Integrates the results of InterProScan runs for up to two protein domain databases</li>
<li>Produces a full set of spreadsheet compatible results including summary statistics on the assembly and resulting annotations</li>
</ul>

<p>1) Diamond is a really fast sequence-search algorithm which can compare nucleotides sequences to protein sequences for very large datasets. Alternatively blastx can also be used for small datasets.</p>

<p>Diamond download: <a href="http://github.com/bbuchfink/diamond/releases/download/v0.7.12/diamond-linux64.tar.gz">http://github.com/bbuchfink/diamond/releases/download/v0.7.12/diamond-linux64.tar.gz</a></p>

<p>Alternatively, blastx can be used for smaller datasets where diamond is slower than blastx:</p>

<p>blastx download: <a href="ftp://ftp.ncbi.nlm.nih.gov/blast/executables/blast+/LATEST/">ftp://ftp.ncbi.nlm.nih.gov/blast/executables/blast+/LATEST/</a></p>

<p>2)  Blast2GO provides a full-featured GUI experience for sequence annotation.  Here, we circumvent some of the limited search options and slow network speeds by limiting its use to Gene Ontology aquisition.  Blast2GO will be run as an indepedent application during the workflow to generate a single output file.  </p>

<p>BLAST2GO download: <a href="https://www.blast2go.com/blast2go-pro/download-b2g">https://www.blast2go.com/blast2go-pro/download-b2g</a></p>

<p>3)  InterProScan is a sequence comparison tool for the identification of protein domains and wraps around several public protein domain databases.  It is used here to identify those domains and also Ontology terms associated with them.</p>

<p>InterProScan Download: <a href="https://www.ebi.ac.uk/interpro/download.html;jsessionid=B76DDDA8BCBB1AD8AFA31F3FE0E476B5">https://www.ebi.ac.uk/interpro/download.html;jsessionid=B76DDDA8BCBB1AD8AFA31F3FE0E476B5</a></p>

<p>The diagram above shows the order in which the seperate operations must be performed in order to generate a complete annotation report.</p>

<p>Steps:</p>

<p>1) Run your preferred search application (diamond or blastx) to obtain a correctly formatted set of search results from your database.  A total of three database runs can be integrated in the current software version.  The Blast6out tab-delimited text output option is required as is the original FASTA database used .</p>

<p>Execution example for diamond:</p>

<p>For making diamond database file:</p>

<pre><code>diamond makedb --in fastadatabase.fasta -d databasename</code></pre>

<p>For running diamond, use the following syntax:</p> 
<pre><code>diamond blastx -d /path_to_diamond_database/databasename -q query.fasta -p no_of_threads -a diamond_alignment_archive.daa -t /directory_for_temp_files --sensitive --evalue evalue_score</code></pre>

<p>For obtaining results in blast6 format, use the following command:</p>
<pre><code>diamond view -a diamond_alignment_archive.daa -o outputfilename.txt -f tab</code></pre>


<p>Execution example for blastx:</p>

<p>For making the blastx database file, use the following command:</p>

<pre><code>makeblastdb -in fastadatabase.fasta -out fastadatabase.db -dbtype prot</code></pre>

<p>For running blastx, use the following command:</p>

<pre><code>blastx -query query.fasta -db fastadatabase.db -out outputfilename.txt -outfmt 6 -num_threads no_of_threads -evalue evalue_score</code></pre>

<p>2) Run InterProScan to generate your InterProScan results file.  This can be run for up to 2 databases (specified by the appl flag).</p>

<p>Execution example for InterProScan:</p>

<p>Note: $input is binded to the configuration files for your interproscan run (refer to: <a href="https://code.google.com/p/interproscan/wiki/RC4HowToRun">https://code.google.com/p/interproscan/wiki/RC4HowToRun</a>)</p>

<pre><code>interproscan.sh -i $input -o $output -f tsv -appl pfam, Panther -goterms -iprlookup
</code></pre>

<p>3) Prepare a configuration file with the proper paths and preferences to run the report_generator.py script.  Details on the configuration file are in the section below.</p>

<p>This script currently supports 2 different sequence search applications, namely diamond and blastx.</p>

<p>When running these applications, it is important to use the blast6 format.</p>

<p>The correct format looks like this:</p>

<p>The current version of the script does not support indexed versions of databases so the flat file (multi-FASTA version) of the database must be supplied for each database queried.</p>

<p>The minimum E-Value can be input in either scientific notation (1e-5) or as a decimal (0.00001).</p>

<p>The minimum coverage requirement is a value between 0 and 1 where 70% coverage represented as 0.7 is the default value.</p>

<p>This script currently has three optional flat file databases containing genus and species information of organisms that could be filtered as potential contaminants in Eukaryote transcriptome studies:</p>

<ul>
<li>Insects</li>
<li>Bacteria</li>
<li>Fungi</li>
</ul>

<p>In in configuration file, each of these can be toggled by writing either "y" or "yes" to include them when searching for contaminants, or write "no" or leave it blank to ignore these filters.</p>

<p>The update.py script will automatically download all of the contaminant files required to run report_generator.py</p>

<p>The files will be deposited into a folder titled "contaminant_databases".</p>

<p>correct update.py usage:</p>

<pre><code>python update.py
